# Growth Metrics

Track these numbers manually during the first month. The goal is to learn which channels produce useful users, not just empty traffic.


## Interpretation Rules

- High GitHub visitors but low stars: README or positioning is not convincing enough.
- High stars but low Marketplace installs: install CTA is weak or Marketplace page is not ready.
- High installs but low reviews: ask real users for feedback after solving their issue.
- Repeated questions: update README, screenshots, or FAQ.
- Repeated bug reports: ship a patch before doing more promotion.

## First Month Targets

These are reference targets for a first open-source project:

- 50-200 Marketplace installs.
- 20-80 GitHub stars.
- 5-15 useful feedback items.
- 1-3 real Marketplace reviews.
- 1-2 patch releases.

Do not treat targets as proof of success or failure. The useful outcome is learning which audience cares about the problem.
